## 执行说明

### 第一阶段：检视阅读步骤

严格按以下顺序执行：

1. **有系统的略读**
   - 书名页 + 序言
   - 目录页
   - 索引（如有）
   - 出版者介绍
   - 挑相关章节看开头结尾
   - 随意翻阅，念几段

2. **粗浅阅读**
   - 从头到尾通读一遍
   - 遇到不懂的地方不要停，继续读
   - 跳过难点，获取整体印象

### 第二阶段：分析阅读步骤

**第一阶段：找出一本书在谈什么（规则1-4）**
- 规则1：分类
- 规则2：用简短句子概括整书
- 规则3：列出重要部分，拟定纲要
- 规则4：找出作者要解决的问题

**第二阶段：诠释内容（规则5-8）**
- 规则5：找出关键字，与作者达成共识
- 规则6：抓出重要句子和主旨
- 规则7：架构论述
- 规则8：确定作者已解决/未解决的问题

**第三阶段：评论（规则9-11）**
- 规则9：未了解前不轻易批评
- 规则10：区分知识与个人观点
- 规则11：批评的角度

### 输出顺序
1. 核心观点（最先输出）
2. 检视阅读结果
3. 分析阅读结果

### 导出功能
分析完成后，将完整报告导出为Markdown格式文件：

**导出规则：**
- 文件命名格式：`《{书名}》阅读分析报告.md`
- 默认保存路径：与原文件相同目录（若原文件可获取路径）
- 若无法确定路径，保存到用户桌面

**导出时机：**
- 分析报告全部输出完成后自动触发导出
- 导出完成后告知用户文件保存路径

**文件编码：** UTF-8

## 批量分析模式

### 触发条件
当用户提供的路径为**文件夹路径**时，自动进入批量分析模式。

### 支持的文件类型
- PDF文件（.pdf）
- Word文档（.docx）
- Markdown文件（.md）
- 纯文本文件（.txt）
- HTML文件（.html）

### 执行规则

**文件发现：**
- 扫描文件夹内所有支持的文件类型
- 按文件名**升序排列**（数字 → 字母 → 中文）
- 忽略子文件夹（仅扫描当前目录）
- 忽略隐藏文件和系统文件

**分析执行：**
- 逐一分析，每个文件独立执行完整的检视阅读 + 分析阅读流程
- 严格按顺序进行，分析完一个文件后再分析下一个
- 互相之间不干扰，每份分析报告完全独立
- 前一份分析不影响后续分析

**输出规则：**
- 每个文件分析完成后立即输出该文件的报告
- 批量模式下，报告**不自动导出为文件**（避免大量文件）
- 全部完成后输出汇总信息

**汇总信息（全部完成后输出）：**
| 项目 | 内容 |
|------|------|
| 分析文件总数 | X |
| 成功分析 | X |
| 分析失败 | X（如有） |
| 失败文件列表 | 文件名 + 失败原因（如有） |

### 汇总导出（如用户要求）
- 仅在用户明确要求时，才将所有报告汇总导出
- 汇总文件命名：`批量分析报告_文件夹名.md`
- 汇总文件保存路径：与原文件夹相同目录

### 示例
```
用户：/tkk-analysis C:\Users\xxx\Desktop\案件资料

系统：
1. 扫描文件夹，找到5个支持的文件
2. 按顺序逐一分析：
   - 分析 01.md → 输出报告
   - 分析 02.pdf → 输出报告
   - ...
3. 全部完成后输出汇总信息
```
