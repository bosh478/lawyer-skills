# tkk-case (v13)

> 全流程法律案件分析技能：沟通启动 → 厘清案件 → 分析 → 文章 → 配图 → vault入库

---

## 核心功能

1. **沟通启动**：支持任意方式描述案件（口述/粘贴文字/发送文件），不以交文件为起点
2. **沟通确认**：在关键节点与用户确认方向和偏好
3. **问题分析**：调用 `tkk-legal-ingest` 分析问题，必要时用 `web-access` 检索网络资源
4. **生成文章**：按结构化模板生成法律分析文章
5. **生成配图**：分析文章段落选择配图候选 → **生成前询问用户确认** → 调用 baoyu-imagine 生成配图
6. **存入vault**：文章存入 `8输出/tkk-case/`、图片存入 `8输出/tkk-case/images/`

---

## 分析框架体系

### 四类案件对应框架

| 案件类型 | 分析框架 | 核心方法 |
|---------|---------|---------|
| 单一刑事案件 | **两阶层四步法** | 客观违法→客观阻却→主观责任→主观阻却 |
| 单一民商事案件 | **请求权基础分析** | 固定请求→检索请求权→分解要件→涵摄归入 |
| 单一行政案件 | **逆向三步法** | 锁定结果→追溯依据→对照证据 |
| 刑民交叉案件 | **双轨并行分析** | 刑事轨道+民事轨道同步/协调 |

### 框架一：两阶层四步法（刑事案件）

**适用**：纯刑事案件（职务侵占、诈骗、盗窃等）

**四步递进审查**（不可跳过/颠倒顺序）：

#### 第一步：客观违法要件审查（构成要件符合性）

**审查要素**（按顺序）：
1. **行为主体**：实施危害行为的人
2. **危害行为**：刑法所禁止的、具有法益侵害危险的身体动静
3. **行为对象**：危害行为直接作用的人或物
4. **危害结果**：行为对刑法保护法益造成的实际损害或现实危险
5. **因果关系**：危害行为与危害结果之间是否存在引起与被引起的客观联系

**操作要点**：
- 完全基于客观证据（物证、书证、鉴定意见、勘验笔录等）
- 忽略行为人主观想法，仅作形式符合性判断
- 重点审查因果关系是否具有法律上的相当性

**输出模板**：
```
【第一步输出：客观要件审查】
1. 行为主体：[是否符合某罪主体要求]
2. 危害行为：[是否具有法益侵害性]
3. 行为对象：[是否明确指向保护对象]
4. 危害结果：[是否造成现实损害/危险]
5. 因果关系：[是否具有相当性]
结论：[符合/不符合] 某罪名客观要件
```

#### 第二步：客观阻却事由审查（违法阻却事由）

**审查事由**：
1. **正当防卫**：针对正在进行的不法侵害实施的必要反击
2. **紧急避险**：为保护较大法益而不得已损害较小法益
3. **被害人承诺**：在被害人有权处分的法益范围内，经其真实有效同意

**操作要点**：
- 此步骤为重要的"出罪安全阀"
- 一旦成立，行为即被正当化

**输出模板**：
```
【第二步输出：违法阻却事由审查】
1. 正当防卫：[是否成立]
2. 紧急避险：[是否成立]
3. 被害人承诺：[是否成立]
结论：行为[被正当化/仍需进一步审查]
```

#### 第三步：主观责任要件审查（故意、过失与认识错误）

**审查内容**：
1. **犯罪故意**：直接故意（希望发生）或间接故意（放任发生）
2. **犯罪过失**：疏忽大意的过失或过于自信的过失
3. **无罪过事件**：主要指意外事件（不能预见的原因引起）
4. **事实认识错误**：行为人对构成要件事实的认识与实际不一致

**操作要点**：
- 结合口供、证人证言、客观行为表现等证据推断主观心态
- 以客观事实为基础验证主观推断

**输出模板**：
```
【第三步输出：主观责任要件审查】
1. 犯罪故意：[直接故意/间接故意/无]
2. 犯罪过失：[疏忽大意/过于自信/无]
3. 无罪过事件：[是否成立]
4. 事实认识错误：[是否影响故意认定]
结论：[有/无] 主观过错
```

#### 第四步：主观阻却事由审查（责任阻却事由）

**审查事由**：
1. **责任年龄**：是否达到法定刑事责任年龄
2. **责任能力**：是否具备辨认控制能力（如精神状况）
3. **违法性认识可能性**：是否确实无法认识到行为违法
4. **期待可能性**：在行为时具体情境下，能否期待其做出合法行为

**输出模板**：
```
【第四步输出：责任阻却事由审查】
1. 责任年龄：[是否达标]
2. 责任能力：[是否具备]
3. 违法性认识：[是否可能]
4. 期待可能性：[是否具备]
结论：行为人[可/不可]进行刑事谴责
```

### 框架二：请求权基础分析（民商事案件）

**适用**：单一民商事案件（合同纠纷、侵权纠纷、物权纠纷等）

**四步操作流程**：

#### 步骤一：固定请求（明确分析对象）

```
要求：
  - 主体明确：请求权人（原告）vs. 请求权相对人（被告）
  - 内容具体：给付的种类、数量、计算方式、责任形式
  - 时间明确：履行期限或计算时点
操作：
  1. 提取请求中的模糊表述
  2. 通过询问或解释予以澄清
  3. 形成明确的"谁向谁主张什么"的请求陈述
```

#### 步骤二：检索请求权基础（"找法"过程）

**检索顺序**（按民法体系）：
1. **契约上请求权** — 意思自治优先
2. **类似契约关系请求权** — 缔约过失、无权代理
3. **无因管理请求权** — 准契约性质
4. **物权请求权** — 物权优先于债权
5. **不当得利请求权** — 衡平救济
6. **侵权损害赔偿请求权** — 一般性救济
7. **其他法定请求权** — 特别法规定

#### 步骤三：分解要件与考察抗辩

```
【请求权基础规范】《XXX法》第XX条
├── 构成要件A：[要件描述]
├── 构成要件B：[要件描述]
├── 构成要件C：[要件描述]
└── 法律效果：[支持何种请求]

抗辩类型：
1. 权利障碍抗辩：合同无效、可撤销、行为能力欠缺
2. 权利消灭抗辩：清偿、抵销、免除、混同
3. 权利阻止抗辩：同时履行抗辩、不安抗辩、诉讼时效抗辩
4. 其他抗辩：不可抗力、受害人过错、第三人原因
```

#### 步骤四：涵摄归入与裁判

```
大前提（法律规范）：《XXX法》第XX条规定，[构成要件]
小前提（案件事实）：经查明/当事人主张，[具体事实]
涵摄判断：
  □ 要件A：事实[编号]是否满足？
  □ 要件B：事实[编号]是否满足？
  □ 要件C：事实[编号]是否满足？

结论：
IF (所有积极要件满足) AND (无有效抗辩成立)
THEN 请求权成立
ELSE 请求权不成立或行使受阻
```

### 框架三：逆向三步法（行政案件）

**适用**：单一行政案件（行政处罚、行政许可、行政强制等）

**三步递进分析**：

#### 第一步：锁定并审查"行政结果"

```
识别维度：
1. 行政行为类型：
   - 行政作为：处罚决定、许可决定、确认决定、强制措施等
   - 行政不作为：对履职申请无回应、拒绝履行、不完全履行
2. 确认行为载体：
   - 作为案件：查找并确认行政决定文书
   - 不作为案件：确认申请材料及行政机关回应状态
3. 明确法律效果：
   - 行政行为产生的权利义务变化
   - 行政相对人受影响的具体权益

审查要点：
- 行政行为是否真实存在
- 行为内容是否明确具体
- 是否属于可诉的行政行为范畴

输出要求：
- 用一句话精确定义待审查的行政行为
- 注明行为作出的时间、机关、文号等关键信息
```

#### 第二步：逆向追溯并审查"法律依据"

```
查找依据层级：
- 法律
- 行政法规
- 地方性法规
- 部门规章
- 地方政府规章
- 其他规范性文件

审查三个维度：
1. 主体依据：行政机关的职权来源条款
2. 实体依据：行政行为内容的法律依据
3. 程序依据：行政行为应遵循的程序规定

归纳合法要件：
- 从有效依据中提取行政行为合法的具体构成要件
- 按主体、实体、程序分类整理要件清单

审查要点：
- 依据是否现行有效
- 依据是否适用于本案情形
- 是否存在法律冲突需要协调
- 如无有效依据，可直接得出违法结论

输出：制作"合法要件清单表"，标注每个要件的法律依据出处
```

#### 第三步：对照审查"事实证据"

```
证据分类整理：
- 主体资格证据
- 实体事实证据
- 程序履行证据

要件对照分析：
- 将证据与第二步归纳的要件逐一对照
- 判断证据是否满足每个要件
- 分析证据的关联性、合法性、真实性

证明责任分配：
- 坚持由行政机关承担主要证明责任
- 审查证据是否形成完整证明链条

输出：制作"要件-证据对照分析表"
```

**行政不作为案件的特殊规程**：
```
审查步骤调整：
1. 审查申请结果：确认申请人是否提出过履职申请
2. 审查法定职责：确定行政机关对该事项的职责范围
3. 审查履职标准：明确履行职责的程序与实体要求
4. 审查履行状态：判断已履行/未履行/不完全履行
```

### 框架四：双轨并行分析（刑民交叉案件）

**适用**：刑民交叉案件（合同诈骗、集资诈骗、职务侵占等）

#### 双轨并行结构

```
┌─────────────────────────────────────────┐
│            刑民交叉双轨并行分析           │
├──────────────────┬──────────────────────┤
│     刑事轨道      │       民事轨道       │
├──────────────────┼──────────────────────┤
│ 两阶层四步法      │ 请求权基础分析       │
│ 构成要件审查      │ 合同效力分析         │
│ 违法性评价        │ 物权/债权分析        │
│ 有责性评价        │ 损害赔偿计算        │
├──────────────────┼──────────────────────┤
│      程序选择建议                        │
├──────────────────────────────────────────┤
│ • 刑民并行：同时推进刑事与民事程序        │
│ • 民先刑后：民事关系影响刑事定性         │
│ • 刑先民后：刑事程序结果影响民事责任     │
└─────────────────────────────────────────┘
```

#### 刑民交叉点识别

```
识别维度：
1. 同一事实是否产生刑民两种法律后果？
2. 刑事判决是否影响民事法律关系？
3. 民事行为是否是刑事犯罪的前提？
4. 涉案财产是否涉及刑民双重返还？
```

#### 程序选择决策树

```
├── 是否涉及人身自由？
│   └── 是 → 刑事优先
├── 是否涉及合同效力？
│   └── 是 → 需要判断合同效力对刑事定性影响
├── 涉案财产是否特定？
│   └── 是 → 可民先刑后
└── 被害人是否选择？
    └── 是 → 尊重当事人选择
```

#### 法律适用协调原则

```
1. 刑事优先原则：刑事认定优先于民事判断
2. 独立性原则：民事法律关系独立评价
3. 一致性原则：刑民结论不得矛盾
```

## 工作流程

### Step 0 — 沟通启动（起点）

**触发方式**：
- 用户口述案件事实
- 用户粘贴文字描述
- 用户发送文件

**行为**：
1. 无论哪种方式进入，先进入**沟通式对话**
2. 通过漏斗式提问逐步厘清案件：涉及人员、行为模式、金额、时间节点等
3. 必要时引导用户提供关键信息

**沟通原则**：
- 开放式提问开启对话，收集广泛信息
- 漏斗式追问逐步聚焦核心事实
- 封闭式提问确认案件类型

---

**读取文件**（当用户提供文件时）：
- 读取 `case_file` → 提取关键事实
- 读取 `case_file2`（如提供）→ 提取对比案例信息

### 维度二：信息校验机制

**操作要求**：对提取的金额和法律数字进行自动校验，确保准确性。

**校验流程**：

```
1. 【金额校验】
   素材金额 → 对比 vault 数额标准 →
   如不符，标注差异并说明

2. 【量刑档次校验】
   涉案金额 → 检索 vault 数额标准 →
   确认量刑档次 → 标注法律依据出处

3. 【证据链校验】
   证据清单 → 检查是否形成完整证明链 →
   标注缺失环节
```

**校验输出模板**：

```
【信息校验结果】

引用前校验：
1. 检查 vault 文件是否存在

4. 如格式错误 → 修正后继续
5. 引用结果记录到引用清单
```

**引用清单格式**：

```

### Step 4 — 生成配图

#### 配图生成流程

```
Step 1: 分析文章内容
   → 读取生成的 markdown 文章
   → 识别核心论点

Step 2: 评估可视化价值
   → 时间线类内容（行为发展、资金流向）→ 优先配图
   → 对比类内容（刑民交叉、要件对照）→ 优先配图
   → 流程类内容（框架步骤、程序流程）→ 优先配图
   → 数据类内容（金额、比例）→ 优先配图

Step 3: 确定配图数量
   → 核心论点 2-3个 → 配图 2-3张
   → 核心论点 4-5个 → 配图 4-5张
   → 复杂案件可适当增加

Step 4: 生成 prompt 并调用 baoyu-imagine
Step 5: 用户确认
Step 6: 插入 Markdown 引用
```

#### 配图优先级

| 优先级 | 内容类型 | 配图 Type |
|-------|---------|-----------|
| 1 | 框架分析核心论点 | framework |
| 2 | 案件时间线/资金流向 | timeline |
| 3 | 刑民对比/要件对照 | comparison |
| 4 | 金额数据/量刑建议 | infographic |
| 5 | 关系图谱/法条结构 | framework |

#### 配图 Type 选择标准

| Type | 中文名 | 适用场景 |
|------|--------|---------|
| `flowchart` | 流程图 | 分析框架步骤、程序流程、两阶层四步 |
| `comparison` | 对比图 | 刑民对比、要件对照、类案对比 |
| `framework` | 框架图 | 构成要件、请求权体系、犯罪论体系 |
| `timeline` | 时间线 | 案件时间节点、行为发展过程、资金流向 |
| `infographic` | 信息图 | 金额数据、涉案比例、量刑档次 |

#### 统一配色方案（极简风格）

| 配色 | 值 | 说明 |
|-----|---|------|
| 背景色 | 白色 #FFFFFF 或浅灰 #F5F5F5 | 简洁明亮 |
| 主色调 | 深灰 #333333 | 文字和主要元素 |
| 辅助色 | 中灰 #666666 | 次要元素 |
| 强调色 | 蓝色 #2196F3 | 重点标注 |

**极简风格特点**：
- 大面积留白
- 少量色块
- 简洁线条
- 清晰层次

#### prompt 生成模板

```
prompt 结构：
"Minimalist legal infographic, white background, Chinese text '[核心论点一句话]', [视觉元素], clean and professional style"

示例：
"Minimalist legal infographic, white background, Chinese text '两阶层犯罪论体系：客观违法→主观有责', key visual elements, clean professional style"

"Minimalist legal infographic, light gray background, Chinese text '请求权检索顺序：契约→物权→侵权→不当得利', key visual elements, clean professional style"

"Minimalist legal infographic, white background, Chinese text '行政行为合法性三步审查：结果→依据→证据', key visual elements, clean professional style"
```

#### 调用 baoyu-imagine

**⚠️ 重要：必须使用 wrapper 脚本执行**

```bash

**禁止直接调用** `bun ~/.claude/skills/baoyu-imagine/scripts/main.ts`，否则环境变量无法正确加载。

**baoyu-imagine 配置检查**：
1. 确认 `~/.baoyu-skills/.env` 存在且包含 `MINIMAX_API_KEY` 和 `MINIMAX_BASE_URL`

3. 详情参见 `~/.baoyu-skills/baoyu-imagine/TROUBLESHOOTING.md`

**配图参数**：
- **格式**：默认 `.jpg`（可用户指定 `.png`）

## 错误处理

| 错误 | 处理方式 |
|------|---------|
| 素材文件不存在 | 报错并终止，提示用户检查路径 |
| vault引用文件不存在 | 警告后继续，跳过wikilink |
| 法律数字与vault不符 | 以vault wiki为准，标注差异并说明 |
| vault无答案 | **必须调用 web-access 检索网络资源**，不得跳过 |
| baoyu-imagine 失败 | 使用 wrapper 重试最多3次；确认 `.env` 配置正确 |
| `invalid api key` | 检查 `~/.baoyu-skills/.env` 中的 `MINIMAX_API_KEY` 和 `MINIMAX_BASE_URL` |
