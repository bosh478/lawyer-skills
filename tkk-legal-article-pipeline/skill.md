# tkk-case-pipeline (v2)

> 全流程法律案件分析技能：素材 → 分析 → 文章 → 配图 → vault入库

---

## 核心功能

1. **接收任意素材**：支持任意案件素材文件输入（不限来源、不限格式）
2. **问题分析**：调用 `tkk-legal-ingest` 的分析逻辑，基于 wiki 知识库进行问题分析
3. **生成文章**：按结构化模板生成法律分析文章
4. **生成配图**：调用 `svg-article-illustrator` 生成5张PNG插图
5. **存入vault**：文章存入 `wiki/问答/`、图片存入 `wiki/问答/images/`

---

## 输入参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `case_file` | string | ✅ | 案件素材文件路径（任意文件） |
| `case_file2` | string | ❌ | 第二个案件素材文件路径（如有） |
| `author` | string | ❌ | 作者，默认"汤康康" |
| `article_title` | string | ❌ | 文章标题，默认从素材提取 |

**输入灵活性**：
- 支持任意格式的法律案件素材（新闻报道/判决书/调研报告等）
- 支持单个或多个文件输入
- 自动识别素材类型并匹配合适的分析框架

---

## 工作流程

### Step 1 — 接收并解析素材

**读取文件**：
- 读取 `case_file` → 识别案件类型（职务侵占/诈骗/盗窃等）
- 读取 `case_file2`（如提供）→ 提取对比案例信息

**调用 tkk-legal-ingest 进行问题分析**：
1. 读取 vault wiki 相关页面，提取相关法律知识
2. 基于 wiki 知识库进行法律问题分析
3. 识别涉及的罪名、法条、司法解释
4. 校验法律数字准确性（数额标准、量刑档次等）

**vault知识库查引用**（Read-Only）：

| vault 文件 | 用途 |
|-----------|------|
| `wiki/concepts/` | 相关罪名概念页 |
| `wiki/entities/` | 相关司法解释 entity 页 |
| `wiki/syntheses/` | 相关案例分析 synthesis 页 |
| `wiki/summaries/` | 相关律师业务指引 summary 页 |

### Step 2 — 生成结构化文章

**分析框架**：
- 根据案件类型，自动选择两阶层/三阶层分析框架
- 根据涉及罪名，调用 wiki 知识库进行构成要件分析
- 根据请求权基础思维，进行善意取得/追赃条件分析

**文章结构**：

```markdown
# {文章标题}

**作者：{author}**

---

## 引言：{案件概述}

[基于素材提取的关键事实]

## 一、案情对比：{本案与类案的相似性}

[类案对比表格]

## 二、案例先行：{相关裁判要旨}

[直接引用相关案例的裁判要旨原文]

## 三、{涉案款项/行为}是否构成赃款/犯罪的认定标准

[两阶层犯罪论体系分析 / 三阶层分析]

## 四、{追赃/定罪}的法律条件分析

[善意取得制度分析 + 请求权基础思维]

## 五、本案的特殊性与{追回/定罪}可行性分析

[与类案关键差异 + 建议策略]

## 结语

[总结 + 行业警示]
```

**必须引用的 wiki 页面**（通过 wikilink）：
- `[[concept_相关罪名]]` — 构成要件分析
- `[[entity_相关司法解释]]` — 法律依据
- `[[synthesis_相关案例分析]]` — 类案参考
- `[[Court_入库案例_案号]]` — 裁判要旨

---

### Step 3 — 生成配图（调用 svg-article-illustrator）

使用 `svg-article-illustrator` 的 `scripts/svg2png.js` 生成5张PNG：

**图片列表**（通用模板）：

| 序号 | 文件名 | 内容描述 |
|------|--------|---------|
| 01 | 打赏追赃-01.png | 标题页（深蓝+金色，专业法律风格） |
| 02 | 打赏追赃-02.png | 案情对比表（本案 vs 类案） |
| 03 | 打赏追赃-03.png | {罪名/制度}法律框架图 |
| 04 | 打赏追赃-04.png | {追赃/定罪}条件分析 |
| 05 | 打赏追赃-05.png | {追回/定罪}可行性评估 |

**动态适配**：
- 图片内容根据案件类型动态调整
- 文件名中的"打赏追赃"替换为案件关键词
- 配色方案根据案件性质调整（刑事/民事/行政）

**生成流程**：
1. 写入 SVG 文件到 `wiki/问答/images/{案件关键词}-XX.svg`
2. 执行 `node scripts/svg2png.js <svg_path> <png_path> 600`
3. PNG 输出到同目录

---

### Step 4 — 插入配图引用

在文章适当位置插入 markdown 图片引用：

```markdown
<!-- 配图：文章标题页 -->
![](images/{案件关键词}-01.png)

<!-- 配图：案情对比表 -->
![](images/{案件关键词}-02.png)

<!-- 配图：法律框架图 -->
![](images/{案件关键词}-03.png)

<!-- 配图：条件分析 -->
![](images/{案件关键词}-04.png)

<!-- 配图：可行性评估 -->

```

## 案件类型自动识别

| 素材特征 | 识别的案件类型 | 分析框架 |
|----------|---------------|---------|
| 出纳/职务便利/挪用资金 | 职务侵占/挪用资金 | 两阶层犯罪论 + 善意取得 |
| 诈骗/欺诈/虚构事实 | 诈骗罪 | 诈骗罪构成要件 + 刑民交叉 |
| 盗窃/秘密窃取 | 盗窃罪 | 盗窃罪构成要件 + 数额标准 |
| 借款/借贷/合同 | 合同诈骗/民事纠纷 | 刑民交叉分析 + 合同效力 |
| 网络打赏/直播/充值 | 网络直播打赏型犯罪 | 善意取得 + 追赃条件 |

---

## 错误处理

| 错误 | 处理方式 |
|------|---------|
| 素材文件不存在 | 报错并终止，提示用户检查路径 |
| vault引用文件不存在 | 警告后继续，跳过wikilink |
| 法律数字与vault不符 | 以vault wiki为准，标注差异并说明 |
| SVG生成失败 | 跳过该图片，继续生成其他图片 |
| PNG转换失败 | 保留SVG文件，用SVG引用作为备选 |
